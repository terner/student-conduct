# 03 — โครงสร้างโรงเรียน

## ลำดับชั้น

```
ปีการศึกษา (academic_years)
  └── ระดับชั้นการศึกษา (education_stages) — อนุบาล, ประถม, มัธยมต้น, มัธยมปลาย
       └── ชั้นปี (grade_levels) — อ.1, ป.1-ป.6, ม.1-ม.6
            └── ห้องเรียน (classrooms) — ป.1/1, ป.1/2, ม.1/1, ม.1/2
```

## การจัดการ

### ปีการศึกษา (`/settings/academic-years`)

- สร้างปีการศึกษาใหม่ (เช่น 2569)
- กำหนด `is_current = true` สำหรับปีปัจจุบัน
- ดูสถานะปีการศึกษา — แจ้งเตือนเมื่อใกล้หมดปี

**กฎ:** `is_current = true` มีได้ปีเดียวเท่านั้น
- Login นักเรียน → ตรวจจาก `is_current = true` เสมอ
- ผู้ใช้เลือกดูข้อมูลปีอื่นผ่าน top bar ได้ แต่แก้ไขไม่ได้
- ขึ้นปีใหม่ใช้ action "ขึ้นปีการศึกษาถัดไป" เท่านั้น

### ระดับชั้นการศึกษา (`/settings/education-stages`)

| Code | ชื่อ | หมายเหตุ |
|------|------|---------|
| `kindergarten` | อนุบาล | |
| `primary` | ประถมศึกษา | ป.1-ป.6 |
| `secondary` | มัธยมต้น | ม.1-ม.3 |
| `highschool` | มัธยมปลาย | ม.4-ม.6 |

- `code` ใช้ภายใน system — เปลี่ยนทีหลังไม่ได้
- `name_th` แสดงใน UI และรายงาน
- `sort_order` ควบคุมลำดับการแสดงผล

### ชั้นปี (`/settings/grade-levels`)

- อยู่ภายใต้ `education_stage`
- มี `level_no` = ระดับชั้นแนวนอน (ป.1=1, ม.1=7, ม.6=12)
- `is_active` เปิด/ปิดชั้นปีที่ไม่ใช้

### ห้องเรียน (`/classrooms`)

- สร้างโดยเลือกระดับชั้น → ชั้นปี → จำนวนห้อง
- ระบบตั้งชื่ออัตโนมัติ (เช่น "ป.1/1", "ป.1/2")
- ต้องระบุ `academic_year_id` เสมอ
- ห้องเรียนมีครูประจำชั้นและครูที่ปรึกษาได้

## การขึ้นปีการศึกษาใหม่

1. Settings → ปีการศึกษา → "ขึ้นปีการศึกษาถัดไป"
2. ระบบจะ:
   - สร้างปีใหม่ (`is_current = true`)
   - Copy ห้องเรียนจากปีที่แล้ว
   - Copy ครูประจำชั้น/ที่ปรึกษา
   - เปลี่ยนปีเก่าเป็น `is_current = false`
3. หลังจากขึ้นปี → import นักเรียนเข้าห้องใหม่

## DB Tables

| Table | Key |
|-------|-----|
| `academic_years` | `is_current` |
| `education_stages` | `code` (unique) |
| `grade_levels` | FK → `education_stages`, `level_no` |
| `classrooms` | FK → `academic_years`, `education_stages`, `grade_levels` |
