# UI & Component Rules

## shadcn/ui v4 with @base-ui/react (⚠️ IMPORTANT)

- **ใช้ `render` prop แทน `asChild`** — @base-ui/react ไม่มี `asChild`
- ตัวอย่าง: `<SidebarMenuButton render={<Link href={...} />}>`
- ตัวอย่าง: `<DropdownMenuTrigger render={<Button>...</Button>}>`
- ตัวอย่าง: `<DropdownMenuItem render={<Link href={...} />}>`
- ตัวอย่าง: `<TooltipProvider delay={0}>` (ใช้ `delay` ไม่ใช่ `delayDuration`)

## react-day-picker v10

- ใช้ `month_grid` แทน `table` ใน classNames
- ใช้ `nav` แทน `nav_button`, `nav_button_previous`, `nav_button_next`
- ตัวอย่าง: `month_grid: cn("w-full border-collapse", defaultClassNames.month_grid)`

## Zod v4

- `.safeParse()` return `unknown` → ต้อง type assertion: `as Record<string, unknown>`
- `z.literal(true)` รับเฉพาะ argument เดียว — ใช้ `.refine()` สำหรับ error message
- ตัวอย่าง: `z.literal(true).refine(v => v === true, { message: '...' })`

## Code Style

- Thai for all user-facing UI (Sarabun font), English for code
- Server Components by default — Client Components only for interactivity
- Zod schemas for ALL input validation (forms, API, env vars)
- Immutable patterns — spread operator, never mutate
- Loading states: Suspense + loading.tsx + skeleton components
- Empty states: every table/list must show empty state message in Thai
- Error boundaries: global + route-level

## Three User Roles

| Role | Scope |
|------|-------|
| **admin** | Full access — settings, import, audit, all reports, bond documents |
| **teacher** | Record scores for any active student; dashboard/reports scoped to assigned classrooms |
| **student** | Self-view only — own score history, current score |
